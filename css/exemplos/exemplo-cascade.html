Cascata CSS

A "Cascata" determina como o navegador decide qual regra aplicar quando há conflitos.

Ordem de Prioridade (da maior para menor)

  1. Estilos do Usuário com !important
  2. Estilos do Autor com !important
  3. Estilos do Autor (CSS normal)
  4. Estilos do Usuário (CSS normal)
  5. Estilos Padrão do Navegador

Exemplo 1: Ordem das Regras

p { color: blue; }
p { color: green; }
p { color: red; }

Quando a especificidade é igual, a ÚLTIMA regra vence! Por isso este texto é vermelho.

Exemplo 2: !important

Este texto deveria ser laranja (inline), mas !important o torna roxo!

⚠️ Evite !important: Use apenas em casos especiais. Ele quebra a cascata natural e dificulta a manutenção do código.

Exemplo 3: Múltiplas Declarações

Esta caixa foi definida duas vezes no CSS. A segunda declaração sobrescreve a primeira (de lightblue para lightcoral).
💡 Dica: A cascata permite que você defina estilos gerais primeiro e depois faça ajustes específicos. É por isso que chamamos de "Cascading Style Sheets"!
